## 1. Goal + scope

Ship patient-facing mobile UI for the biomarker foundation built in 2.5A/B/C — two screens only: **Dashboard** and **Detail**. Founder visual approval gate. Report view + Upload flow deferred to Phase 2.5E.

### 1.1 In scope (2.5D)

- **Dashboard screen** (`/home/biomarkers/index.tsx`) — greeting, "new report ready" banner, 3-stat strip, category filter pills, scrollable biomarker card list with range bar + sparkline + Δ%.
- **Detail screen** (`/home/biomarkers/[id].tsx`) — WHOOP-style Dial, 6-month AreaChart with optimal band, Reference ranges card, "What it means" clinical note.
- **Viz primitive library** in `packages/mobile-ui/src/biomarker/viz/` — `Dial`, `RangeBar`, `Sparkline`, `AreaChart` ported from `viz.jsx` to `react-native-svg`.
- **Home-screen "New report" banner** — entry point from `/home/index.tsx` when unread `biomarker_report` exists.
- **Convex patient-facing query** — `convex/biomarker/patient/myBiomarkerReports.ts` (session-token auth, PATIENT role).
- **Deep-link handler** — `onlyou://biomarkers/:reportId` routes to Dashboard with scroll-to-marker behavior (for `lab_report_ready` push payloads from 2.5C).
- **Light/dark toggle** — wired to `useColorScheme()` + manual override via profile settings (persists in `AsyncStorage`).
- **New design token** `biomarkerAccent` = `#E8A04C` (dark) / `#B4641F` (light) registered in `@onlyou/core/tokens/colors`. Distinct from consultation `accentWarm` (`#C4956B`).
- **Fonts** — Instrument Serif + JetBrains Mono loaded via `expo-font`, registered as **biomarker-scoped** families (`font-serif-biomarker`, `font-mono-biomarker`). Intentional divergence from DESIGN.md (which specifies Playfair Display for serif). Rationale: editorial/WHOOP aesthetic. Requires founder sign-off before Wave 1 locks. Fallback plan: swap to Playfair Display + no mono if founder prefers project-wide consistency; viz primitives remain unchanged.

## 3. Architectural invariants

1. **Mock-first.** Waves 2+3 use local mock data; no network calls until Wave 4. Protects founder visual approval from backend flakiness.
2. **Viz primitives are theme-agnostic.** Accept `color` / `bg` props; never read theme context directly. Screens inject theme values.
3. **Session-token auth only.** No `ctx.auth.getUserIdentity()`. Patient query reads token from caller args, looks up `sessions` table. Same pattern as 2.5C queries.
4. **No biomarker read surface auto-leaks.** Patient query projects fields explicitly — same defensive shape as `biomarkerReportsForPatient`.
5. **Clinical Luxe compliance.** 24px horizontal padding, floating labels on inputs (n/a in 2.5D), no hardcoded hex (all colors from `@onlyou/core/tokens`).
6. **Single biomarker accent token.** `#E8A04C` lives as `biomarkerAccent` alone — not reused as `accentWarm`, not collapsed into existing `chart-2`. Reconciliation note added to DESIGN.md.

## 8. Risks + mitigations

| Risk                                                          | Mitigation                                                                    |
| ------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| `react-native-svg` Dial renders differently on Android vs iOS | Snapshot-test both platforms in CI; manual device test Wave 4                 |
| Instrument Serif + JetBrains Mono bundle size                 | Both subsetted (Latin only); measure APK delta in Wave 1                      |
| Founder rejects dark-default aesthetic                        | Wave 2 visual review is gate — pivot to light-default before Wave 3 if needed |
| Convex query missing reports due to filter on `deletedAt`     | Match doctor-portal query filter; add test for soft-deleted exclusion         |
