## 3. Architectural invariants

1. **Two-register guardrail.** All biomarker UI imports from `@onlyou/core/tokens/biomarker` only. Clinical Luxe tokens (`theme/colors`) are a lint error inside biomarker paths. Per [[decisions/2026-04-17-biomarker-design-register]].
2. **Mock-first.** Waves 2 + 3 use local mock data; no network until Wave 4. Protects founder visual approval from backend flakiness. Mock fallback retained behind env flag post-Wave-4 for offline demos.
3. **Viz primitives are palette-aware, not theme-aware.** They import `biomarkerPalette` directly — there is no dark mode in 2.5D. When dark mode lands post-foundation, viz primitives switch to taking palette via prop/context.
4. **Session-token auth only.** No `ctx.auth.getUserIdentity()`. Patient query reads token from args, looks up `sessions` table. Same pattern as 2.5C queries and `NEVER use ctx.auth.getUserIdentity()` rule.
5. **No biomarker read surface auto-leaks.** Patient query projects fields explicitly — defensive shape matches `biomarkerReportsForPatient`. Schema changes do not auto-propagate to the wire.
6. **24px horizontal padding.** Bundle uses 20; spec enforces 24 per CLAUDE.md.
7. **No new design tokens created.** Reuse existing `biomarkerPalette`. `#E8A04C` (dark amber) is a _dark-mode_ value — not needed in 2.5D light-only build.

## 4. Data contracts

### 4.1 Patient query shape (Wave 4)

```ts
// convex/biomarker/patient/myBiomarkerReports.ts
export const myBiomarkerReports = query({
  args: { token: v.string() },
  handler: async (ctx, { token }) => {
    const session = await ctx.db
      .query("sessions")
      .withIndex("by_token", (q) => q.eq("token", token))
      .unique();
    if (!session || session.expiresAt < Date.now()) {
      throw new ConvexError({ code: "unauthenticated" });
    }
    const caller = await ctx.db.get(session.userId);
    if (!caller || caller.role !== "PATIENT") {
      throw new ConvexError({ code: "forbidden" });
    }
    // ... same projection as biomarkerReportsForPatient, scoped to caller._id
  },
});
```

Returns `Array<{ report, values[] }>` — same projection as doctor-portal query, scoped to caller's own `userId`. Client transforms to dashboard card shape.

### 4.2 Mock dataset shape (Waves 2 + 3)

```ts
type BiomarkerMock = {
  id: string;
  name: string;
  cat:
    | "Metabolic"
    | "Lipids"
    | "Hormones"
    | "Inflammation"
    | "Vitamins"
    | "Organ"
    | "Blood";
  unit: string;
  value: number;
  low: number;
  high: number;
  optLow: number;
  optHigh: number;
  trend: number[]; // length 7
  status: "optimal" | "watch" | "high" | "low";
  prev: number;
};
```

Port 24 rows verbatim from `data.jsx`.

### 4.3 Hook-to-card transform (Wave 4)

`use-biomarker-reports` maps Convex `values[]` rows to `BiomarkerMock` shape. `canonicalId` → looked up in `biomarker_canonicals` seed to recover name/unit/category. Status derived from `status` field on `biomarker_values`. Trend/prev computed by joining prior reports for same `canonicalId`.

## 8. Risks + mitigations

| Risk                                                          | Mitigation                                                                                     |
| ------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| `react-native-svg` render parity between Android and iOS      | Wave 5 manual device test on both platforms; snapshot tests per primitive in Wave 1            |
| RN SVG does not support `filter: drop-shadow` (used in Dial)  | Replace with opacity-layered duplicate circle for glow effect — documented in Wave 1 Dial port |
| Reanimated v4 API diverges from prototype CSS animations      | Wave 5 ports one animation at a time; drop any that bloat APK > +50KB                          |
| Convex `canonicalId` → category lookup performance            | Join is read-only; cache category map at hook level. Add test to verify single query           |
| Deep-link fires before session hydration                      | Queue in `_layout.tsx`; flush after `useAuthStore.hydrated === true`                           |
| Founder rejects editorial register vs Clinical Luxe unity     | Register already approved 2026-04-17; spec closure not revisitable here. If raised, escalate   |
| `lab_results` directory collides with existing `lab-booking/` | Confirmed distinct; `lab-booking/upload-results*` is future Phase 2.5E surface                 |
