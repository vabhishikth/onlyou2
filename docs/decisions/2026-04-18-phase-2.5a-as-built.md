## Notable deviations from plan (decided during implementation)

- **Feature flag DB key** uses SCREAMING_SNAKE (`BIOMARKER_PARSING_ENABLED`) to match all 7 existing flags; the plan's lowercase `biomarker_parsing_enabled` was descriptive prose.
- **ESLint rules** placed in `apps/mobile/eslint.config.js` (nearest-config wins in flat config), not root `eslint.config.js`.
- **Seeder ‚Üí Convex** path uses `spawnSync` of the Convex CLI bundle rather than `ConvexHttpClient`. Internal mutations are not reachable via the HTTP client by design; this is the only working path on Windows (also avoids the cmd.exe 8191-char command-line limit on the 26 KB JSON payload).
- **Hemoglobin `pregnancySensitive`** corrected from `false` ‚Üí `true` during Task 13 (hemoglobin ranges genuinely differ in pregnancy ‚Äî caught by the validation test).
- **Threshold invariant** in README relaxed from strict `<` to `‚â§` on action bounds because every seed row has `actionBelow == subOptimalBelowMin` and `subOptimalAboveMax == actionAbove`. Documented as a clinical-guideline reality in `packages/core/seeds/README.md`.
