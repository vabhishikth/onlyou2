# Decision — Phase 2.5 scope boundary

**Date:** 2026-04-17
**Phase:** 2.5 brainstorm
**Spec:** [[superpowers/specs/2026-04-17-phase-2.5-biomarker-foundation-design|Phase 2.5 design spec]]

## Decision

Phase 2.5 ships the full cross-vertical biomarker foundation — one patient-facing UI, two backend contracts for other portals, one parse pipeline for all three ingestion surfaces. Specific boundaries:

### In scope

- Patient upload path — full UI end-to-end: pick → parse → classify → render.
- Diagnostic-center (lab) upload path — **backend contract only**. Mutation + lab-order scoping + auth stub. UI lands in Phase 6 lab portal.
- Doctor-ordered path — `lab_orders` table + `biomarkerReportsForPatient` query. Doctor portal UI wires in Phase 4.
- Seed data: **32 markers** spanning union of 5 verticals' ordered panels + CBC essentials + lipid-separated (HDL/LDL/Trigs separate). ~45 reference-range rows after sex/age splits.
- Pregnancy safety guard (see [[2026-04-17-pregnancy-safety-guard]]).
- Curation queue + admin surface + backfill reclassification — **all three as named deliverables**, not just write-path plumbing.

### Out of scope (destination phase named in every case — see [[DEFERRED]])

- Real push notifications → Phase 8.
- Lab portal web UI → Phase 6.
- Doctor portal biomarker UI → Phase 4.
- External lab APIs → Phase 8+.
- Dark mode wiring → post-foundation (tokens defined).
- Long-term line chart across >2 reports → post-foundation.
- Narrative regeneration on backfill → post-foundation.
- PDF export / share-with-external-doctor → post-foundation.
- Patient-written notes on markers → post-foundation.
- Pregnancy + post-menopausal range DATA → PCOS phase (the GUARD ships now).
- Pediatric ranges → post-foundation.
- VoiceOver rotor + haptics → Phase 8.
- Automated visual regression → post-foundation.
- Admin audit log UI + bulk import + role-scoped curation → post-foundation.

## Why the 32-marker target

"Closer to 25" would trigger `unclassified` banners on nearly every first real patient upload and undermine the demo. 32 is the union of five verticals' panels + CBC + separated lipid markers. Designed to cover the realistic coverage of a typical Indian lab report in the target verticals while keeping the seed reviewable in one sitting by a clinical advisor.

## Why curation backfill is a named deliverable

A write-only queue never closes the feedback loop. A curator adding a range must trigger reclassification on affected `biomarker_values` across all patients, recompute envelope counts, and emit a "your report was updated" notification — or the work doesn't compound. This costs the curation admin surface + the background job + a tested idempotent reclassify flow.

## Why backend contracts ship without UIs

Keeps Phase 6 (lab portal) and Phase 4 (doctor portal) from rebuilding biomarker infrastructure. When their UIs arrive, the backend is already tested. But we don't register dead endpoints in prod — `LAB_PORTAL_ENABLED` and `DOCTOR_PORTAL_ENABLED` env flags gate registration. No attack surface, no observability noise, until the flags flip in their destination phases.

## How to apply

- Every scope question that comes up during Phase 2.5 execution gets measured against this decision. Additions require an explicit amendment here + founder approval.
- Every "deferred past 2.5" item must name its destination phase in [[DEFERRED]] before merge. "Later" is never valid.
