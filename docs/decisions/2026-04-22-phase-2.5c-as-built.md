## What shipped vs what the spec said

The spec called for 7 workstreams (real ingestion, smart unknowns, reclassify, notifications, portal contracts, 2.5B carry-forwards, tests). All shipped. Zero scope creep; three documented intentional divergences from the plan's literal text:

1. **Role literals are UPPERCASE** (`"PATIENT"` / `"DOCTOR"` / `"ADMIN"`) throughout, because the `ROLES` enum is uppercase. Plan snippets used lowercase; corrected in-place on every test and fixture.
2. **convex-test module glob is inline** (`import.meta.glob("../../**/*.ts")`) per file, not a `_modules.ts` barrel. Barrel didn't exist and the inline pattern is the established repo convention (see `convex/__tests__/biomarker/intake-upload.test.ts`).
3. **Manual Convex dashboard E2E → automated driver.** Plan Task 32 asked a human to click through the dashboard; the driver `scripts/run-manual-e2e.ts` replaces that entirely. Two reasons: (a) Wave 5 locked `intakeUpload` behind session-token auth so dashboard identity can't invoke it; (b) the founder is non-technical and asked to automate.

## Surprises during execution

### The first live E2E surfaced three Phase 2.5B bugs at once

All three masked by unit-test mocks; all three silent-failure modes that stuck lab_reports in `analyzing` forever because the `markReady` / `markParseFailed` / `scheduleRetry` paths were never reached:

1. **`getUnitConversions` used `await import(...)` inside a V8 query handler.** Convex V8 query runtime rejects dynamic imports. Fix: static top-level JSON import + pre-shaped `UNIT_CONVERSIONS` constant at module load. (`8484b46`)
2. **`parseLabReport` used `Buffer.from(...).toString("base64")` in a V8 action.** Node's `Buffer` is not a V8 global. Fix: `arrayBufferToBase64` helper with chunked `String.fromCharCode.apply` + `btoa` (chunk size 32 KiB). (`85ba690`)
3. **`AutoDraftRangeOutputSchema` used `.nullable()` but Claude returns partial JSON when `is_real_biomarker: false`** (omits keys entirely → `undefined`, not `null`). Zod threw; unhandled throw propagated through `autoGenerateDraftRange` into `parseLabReport` mid-loop. Fix: schema `.nullable()` → `.nullish()`, field-presence checks switched `=== null` → `== null`, and `callAutoDraftRange` wrapped in try/catch returning new outcome `rejected_claude_error`. (`c8c94a7`)

These are the strongest argument for live E2E as a phase gate. Mocked suites can't catch runtime-environment mismatches, and the pipeline's "silent stuck in analyzing" symptom masked the root cause behind the retry cron's re-entrancy.

### The Convex V8 runtime is stricter than Phase 2.5B assumed

Two of the three bugs above (`await import`, `Buffer`) are V8-runtime gotchas. Phase 2.5B built the pipeline correctly per the unit-test fixture environment (which has Node `Buffer` via vitest's polyfills) but wrong for the production Convex deployment. Lesson for future Convex phases: **always run one live E2E before closing the phase**, and add a CI smoke job if feasible.

### Task 32 scope grew into 3 bug fixes + a driver

Originally allotted ~10 min of dashboard clicking. Actual work: driver script + 3 fixes + 4 E2E runs. Still shipped same day. Log separately as a DEFERRED entry so 2.5D can budget for similar scope expansion if further bugs surface.

### Phase review's I-1 caught a real security hazard

`admin.generateUploadUrl` + `admin.getE2EStatus` shipped as public mutation/query gated only by `assertNotProd()`. `getE2EStatus` would have returned any patient's lab data to an attacker with the dev deployment URL + a guessable `labReportId`. Post-review fix converted both to internal, drove via `npx convex run` in the script. Good example of why phase-level review (vs per-task review) catches cross-cutting issues.

## Lessons for 2.5D

1. **Live E2E is a phase gate, not a post-merge courtesy.** Phase 2.5D should ship one real mobile build against a real dev Convex deployment before marking the phase done. Unit tests + Storybook mocks will miss runtime-environment mismatches the same way 2.5B's mocks did here.
2. **Post-review security checks on public surfaces.** Any new `query` / `mutation` / `action` (vs `internalQuery` etc.) needs an explicit justification in its docblock. Next time reviewers can grep for the pattern in one pass.
3. **When `assertNotProd()` is the only gate, assume it's a time bomb.** Convert to internal + drive via CLI unless there's a real need for client access.
4. **`.nullish()` over `.nullable()` for every LLM-emitted field.** Claude's JSON generation is consistent about keys-present-with-null vs keys-omitted only when the schema explicitly forces it. Phase 2.5D mobile forms have similar nullable-vs-nullish risks on form submissions.
5. **Triplicated config is the exact class of bug Wave 5 review tried to prevent.** Hoist-to-shared-package early; two copies is already too many.
