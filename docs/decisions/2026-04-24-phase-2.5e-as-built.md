## Deviations from plan

1. **`collectionDate` typed as `string` (ISO), not `number`.** Schema is `v.optional(v.string())`; plan snippet assumed number. Adapted types across query, hook, and test fixtures. Added normalization helper so AI-extracted dates ingest as `YYYY-MM-DD`.
2. **Count-based `Promise.all` assertion dropped.** convex-test exposes no `ctx.db.get` counter. Test asserts result shape only. Refactor still lands the latency benefit.
3. **`EXPLAINER_MAP` expanded to 31 entries.** Plan estimated ~24. Seed canonical ids use long-form (`ldl_cholesterol`, `hdl_cholesterol`, `triglycerides`, `vitamin_d`, `hba1c`) that did not match the mock short-form keys; added aliases + 6 new full entries (hba1c/mcv/free_t3/free_t4/iron_serum/vitamin_b12). PCOS hormone markers (shbg/fsh/lh/estradiol/prolactin/dhea_s/testosterone/amh) left to fallback — deferred to Phase 6+.
4. **DetailHero → `Dial`, not `RangeBar`.** Plan instruction to thread direction into `DetailHero.tsx` was misaligned with reality; the second RangeBar instance is inside `lab-results/[id].tsx` Reference card ("Your Value" row). Threaded there.
5. **Seed refactor inlined status classification.** `seedOneReport` computes `assignStatus` inline rather than calling `classifyRow` — Convex mutations cannot invoke queries. Risk of drift noted in code review (Minor).
